    Setup
    Pick your Music folder and one EarCrate workspace. EarCrate will create its database, cache, manifests, renders, playlists, and stems inside the workspace. Your Music folder is treated as read-only source material.
    Show derived folders
    These are derived from the workspace. You normally should not edit them.
    Performance
    Lower is faster. 180s is plenty for BPM/key. Raise only if you have long DJ sets or mixes where the useful part starts late.
    Parallel decode + analysis across CPU cores. Auto uses all but two.
    

    

    Add music // multi-folder ingest
    

    Copy music from anywhere (external SSD, downloads, old drives) into the managed library. Sources are never modified; copies land under master/ingested/<batch>/, content-hash deduped, manifest-journaled, rollback-able. Then Organize + Retag builds the clean Artist/Album/NN Title tree with amended tags under working/organized/ — your originals stay verbatim.
    

    


    

    

    

    Reorganize the source // in place
    

    Moves your ORIGINAL files inside the music folder into Artist/Album/NN Title. Unlike Organize+Retag (which copies to a new tree), this rewrites the source layout itself. It is reversible (every move is journaled) and never deletes: anything it can't identify goes to _unsorted/. Always Preview first — nothing moves until you Apply.
